The Vanroll JSON Shim Format

A Vanroll Shim is a pure data structure (JSON) that maps user intentions to DOM manipulations on a specific domain. Because it contains zero executable JavaScript, it can be updated automatically by an LLM without presenting security vulnerabilities.

  • ✓ Decoupled layout data
  • ✓ No executable code
  • ✓ AI-updatable structures 
{
  "domain": "example-shop.com",
  "version": "1.0.0",
  "selectors": {
    "search_input": "input[type='search']",
    "search_submit": "button.search-submit",
    "product_price": ".price-tag >> text"
  },
  "flows": {
    "search_and_get_price": [
      { "action": "type", "target": "search_input", "value": "{{product_name}}" },
      { "action": "click", "target": "search_submit" },
      { "action": "scrape", "target": "product_price", "key": "item_price" }
    ]
  }
}

Component Architecture

1. The Orchestrator

Injected into LLM interfaces (e.g., Gemini, ChatGPT).

Injects system instructions into the prompt context telling the LLM how to format automation tasks. Monitors the live streaming DOM response and relays structured commands.

2. The Bridge

Chrome Extension Background Service Worker.

Acts as the network and session switchboard. Receives intercepted commands, fetches the corresponding JSON Shim, and routes the payload to the target tab.

3. The Action Engine

Injected into Target Web Tab.

A read-only, hardcoded interpreter class. Receives the JSON Shim, maps parameters to the DOM, handles error retry loops, and passes data returns back up the pipeline.

WebMCP Integration

Parses loaded Shim flows and registers them as tools for Native Browser AI utilizing the navigator.modelContext.registerTool API. Allows AI assistants to directly execute Vanroll JSON templates as standard JSON Schema tools.
